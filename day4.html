Various Keys in Database Management System

Keys

A Key is an attribute or a set of attributes, which is used to uniquely identify a record of a table. It uniquely identifies records or a combination of records from huge database tables.

1◈Primary-Key

Primary key is an attribute, which uniquely identifies each record within a table. It can't accept NULL value. We can have only one primary key in a table. A primary key is the column that contains values that uniquely identify each row in a table. A primary key is a special relational database column designated to uniquely identify each table record.

2◈Alternate-Key

It is a column or group of columns in a table that uniquely identify every row in that table. All the remaining candidate key which are note selected are called Alternate Key. A table can have multiple choices for an alternate key but only one can be set as a primary key. All the keys which are not primary key are called alternate keys.

3◈Super-Key

A super key is aa combination of all possible attributes which can uniquely identify the rows(tuples) in a table.

In the given Student Table, we can have the following keys as the super key.
{Roll_no}

All the above kets are able to uniquely identify each row. So, each of these keys are super keys.

4◈Candidate-Key

Candidate key is a minimal super key, which contains no extra attributes. It is also called subset of super key. It is a set of attributes that uniquely identify records in a table. Candidate Key is a super key with no repeated attributes. The primary key should be selected form the candidate keys. Every table must have atleast a single candidate key. A table cna have multiple candidate keys but only a single primary key.

Example:
In the table student(above example) though the super keys {Roll_no}, {Registration_No}, {Roll_no,Registration_No}, {Roll_no,Name}, {Name, Registration_No}, {Roll_No,Name,Registration_No} give us unique value but when we take subset of this key we will get two attributes and this does not satisfy tha minimal super key concept. Hence we have to take either {Roll_No} or {Registration_No} as a candidate key.

5◈Foreign-Key

A foreign key is a field(or collection of fields) in one table that refers to the primary key in another table. A foreign key is a key used to link two tables together.
Example:

In the Student table, the field Stud_Id is a primary key because it is uniquely identifying all other fields of the student table. On the other hand, Stud_Id is a foreign key attribute for the Department table because it is acting as primary key attribute for the Student table. It means that both the student and department table are linked with one another because of the Stud_Id attribute.
